> ⚠️ 本文档内嵌的命盘示例数值为**开发期样例（虚构生辰）**，仅用于说明字段结构与版面布局，并非真实排盘结果，请勿据此对照。

> 设计师参考文档。**给任意视觉工具的输入 brief**。
> 本文档定义"一份命理报告的页面结构"——不是抽象 schema，是带 ASCII 整页布局的具体规范。
> 字段长度、章节顺序、视觉权重都已敲定，**设计师只负责美化，不负责重排信息架构**。

## 设计纪律线（先看完这一节再开始设计）

### ✅ 必做
1. 报告分两层：**上方速览卡（结构化）+ 下方深度分析（半自由文本）**
2. 速览卡的**第一屏**必须包含：Hero + 排盘可视化（紫微 12 宫盘 + 八字四柱盘）+ 命盘核心数据
3. 排盘**不是单独章节**，是速览卡的开头部分，必须紧贴 Hero 之下
4. 三种模式（A 八字 / B 紫微 / C 综合）的速览卡都遵循同一种节奏：Hero → 排盘 → 数据卡 → 强弱卡
5. 配色 / 字体 / 装饰沿用 `UI-DESIGN.md` 中定义的 token
6. 所有字段长度上限**必须用 CSS 兜底**（`text-overflow: ellipsis` 或固定容器高度 + 滚动）
7. 免责声明文本写死，不允许 LLM 改

### ❌ 不要
- 不要把排盘放到附录 / 尾部 / 单独 tab
- 不要为"高级感"删掉强弱卡（这是命理报告的核心）
- 不要让排盘可视化占满首屏（要留 Hero 的呼吸感）
- 不要把六维分析做成等高卡片（事业/财运 会比 子女/健康 长，要适应内容）
- 不要给免责声明加装饰边框（克制）
- 不要用太极八卦图、塔罗装饰、星盘霓虹这类符号

### 输出形态
- **单文件 HTML**（CSS inline，无外部依赖除字体 CDN）
- 桌面优先（1080px），手机 ≥360px 能看
- 打印友好（A4 横版可分页）
- 可截图分享（整页 / 局部都可）

## 速览卡（所有模式共有，是报告的"门面"）

> **关键**：速览卡是用户截图分享的部分，必须**信息密度高 + 视觉冲击强 + 单屏完整**（桌面 1 屏，手机 1.5 屏内）。

### 速览卡 = Hero + 排盘 + 数据卡 + 强弱卡（按这个顺序）

#### 1. Hero 区（顶部，居中）

```json
{
  "archetype_name": "金水盖头的偏财客",         // 3-7 字, 海报式标题
  "axis_oneliner":  "戊土生于亥月，靠庇护立身，借时势成事",   // ≤30 字
  "meta_solar":     "阳历 2000-01-01 09:30 男",
  "meta_lunar":     "农历 辛未年十月初九巳时"
}
```

**视觉规范**：
- archetype_name：大字（48-64px），思源宋体，墨色，居中
- axis_oneliner：副标题（18-22px），灰墨色，居中，**最多 30 字否则截断**
- meta：小字（12-14px），等距字体（数字感强），灰色，居中

#### 2. 排盘可视化区（紧贴 Hero 之下）

按模式不同显示：
- **模式 A**：仅八字四柱盘
- **模式 B**：仅紫微 12 宫盘
- **模式 C**：紫微 12 宫盘 + 八字四柱盘**并列**（桌面左右 / 手机上下堆叠）

**紫微 12 宫盘视觉规范**（4×4 网格，地支方位固定）：
```
巳 | 午 | 未 | 申
辰 |  中宫    | 酉
卯 |  命主    | 戌
寅 | 丑 | 子 | 亥
```

每个宫位含：宫名 + 干支 + 主星（庙旺利平陷）+ 辅星 + 大限范围 + 化曜染色  
**命宫**：朱砂红边框 + "命"印章  
**身宫**：右上角"身"字方印  
**当前大限**：底部高亮  
化曜染色：禄=朱砂 / 权=青蓝 / 科=翠绿 / 忌=玄黑加粗下划线

**八字四柱盘视觉规范**（横向四列）：
```
年        月        日        时
[十神]    [十神]    [日主]    [十神]
天干大字  天干大字  天干大字  天干大字
地支中字  地支中字  地支中字  地支中字
[藏干含十神 stack]
[星运/自坐/纳音]
─────────── 大运带 ───────────
[10 个大运 + 起运岁 + 当前高亮]
```

#### 3. 命盘核心数据卡（4-5 张并排小卡）

```json
{
  "geju":          { "value": "偏财格",     "confidence": "中" },
  "wangshuai":     { "value": "偏弱",       "score": -2.8 },
  "tiaohou":       { "value": "甲、丙" },
  "wuxing_radar":  { "data": { "木": 0.5, "火": 2.5, "土": 3.5, "金": 1.5, "水": 2.0 }, "top": "土", "missing": "木" },
  "key_relations": [
    { "type": "六冲", "items": "巳-亥", "impact": "高" },
    { "type": "盖头", "items": "己亥·戊子", "impact": "高" }
  ]
}
```

**仅模式 C 额外**：两盘主轴速览（八字气泡 + 紫微气泡 + 判定徽章 🟢/🟡/🔴）

#### 4. 强弱卡（左右两栏）

```json
{
  "strengths": [
    { "title": "稳健内核", "desc": "比劫得力，根基扎实" },   // ≤25 字
    { "title": "印星护身", "desc": "时柱丁火滋养日主" },
    { "title": "财气活跃", "desc": "财星透干且月令当令" }
  ],
  "weaknesses": [
    { "title": "盖头重",   "desc": "己亥戊子双盖头，理想现实拉扯" },
    { "title": "调候缺位", "desc": "金水寒滞，火不到位行动力季节性低落" },
    { "title": "巳亥相冲", "desc": "驿马与配偶宫互冲，长期波动" }
  ]
}
```

**视觉规范**：
- 左右两栏，每栏 3 张卡，**固定 3 张不能多不能少**
- 优势卡：墨绿/淡青底色 + ★ 图标
- 劣势卡：灰红/灰色底色 + ✘ 图标
- 卡片**等高等宽**

## 深度分析章节（速览卡下方，半自由文本）

> 这部分允许 LLM 半自由发挥，但**每段有字数上下限**。模板用 CSS 控制段落容器高度。

### 模式 A 深度章节（顺序固定）

1. **六维分析卡**（事业 / 财运 / 婚恋 / 子女 / 六亲 / 健康）
   - 每维 summary（≤30 字 + tag）+ detail（150-260 字）
   - 折叠卡片，默认显示 summary，点击展开 detail
   - **6 张卡，等宽不等高**（detail 长度自然差异）

2. **关键时间节点**（5 个，固定数量）
   - 垂直时间线，圆点 + 卡片
   - 字段：year / age / title / event / severity / desc（≤80 字）
   - severity 五档：高吉/小吉/中性/小凶/高凶/转折，用色彩/图标编码

3. **行动建议清单**
   - 固定 3 类（发展方向 / 人际 / 风险防范）
   - 每类 2-3 条

4. **免责声明**（写死）

### 模式 B 深度章节

1. 命格核心结构（命宫主星 + 生年四化落宫 + 身宫，180-250 字）
2. 三方四正分析（200-300 字 + 四宫位置小图）
3. 大限走向时间轴（当前 ± 1 大限 + 整段时间线）
4. 六维分析卡（同 A1，但用紫微术语）
5. 重点流年清单 🔴🟡⚪（红黄白三档）
6. 行动建议 + 免责声明

### 模式 C 深度章节

1. 主轴印证结论（180-250 字）
2. 阶段印证时间轴（水平双轨 + 重合点高亮）
3. 六维交叉对账（6 张大卡，每卡：八字take / 紫微take / 印证标记 / 融合结论 100-150 字）
4. 冲突清单（按 4 条规则触发的项，0-N 条）
5. 综合定论（终极主轴句 + 5 节点时间线 + 3 风险窗 + 2 优势策略 + 3-5 条建议）
6. 置信度自评（三色标签）
7. 免责声明

# 紫微 + 八字排盘 UI 设计讨论

> ⚠️ 本文档内嵌的命盘示例数值为**开发期样例（虚构生辰）**，仅用于说明视觉与布局，并非真实排盘结果。

> 本文档是 UI 设计的单一讨论入口。所有视觉决策、迭代记录、参考素材、反馈意见都汇总在这里。
> 算法 / 提示词 / Skill 流程在别的文档，此处不展开。

---

## 1. 我们在设计什么

两套 HTML 模板，作为"AI 八字+紫微 Skill"的视觉输出：

- **模板 A：紫微 12 宫盘**（已有 mockup v0：`ziwei-mockup.html`）
- **模板 B：八字四柱盘**（未开始）

最终用法：算法层产出 JSON → 渲染脚本填占位符 → 输出**单文件 HTML** → 用户用浏览器打开/截图分享。

## 2. 视觉方向

### 双轨融合：**现代极简 × 中式水墨**

| 维度 | 现代轨 | 水墨轨 |
|---|---|---|
| 底色 | 微暖白（不刺眼） | 宣纸米色 #f5f1e8 |
| 字体 | 等距英数字（JetBrains Mono / IBM Plex Mono） | 思源宋体 / Noto Serif SC |
| 装饰 | 留白、细线网格 | 装裱双线、印章、晕染 |
| 信息层级 | 字号阶梯 + 颜色弱化 | 主星大字、辅星小字、小字最弱 |
| 装订感 | 卡片化、阴影、卡片间距 | 整版"一张盘"、外裱内挂 |

### 视觉参考方向
- 日本独立杂志（克制、留白、字体张力）
- 蘋果中文官網（宋体+无衬线对比）
- 朱赢椿书籍设计（中式书卷气、不复古不卡通）
- **避免**：羊皮纸质感、太极八卦堆 SVG、Q版插画、塔罗牌风、星盘霓虹风

### 现行配色 token（建议保留语义分区，色值可调）


## 3. 模板 A：紫微 12 宫盘 — 设计要点

### 布局
4×4 网格，外圈 12 格为十二宫，中心 2×2 为"中宫"：
```
巳 | 午 | 未 | 申
辰 | 中宫     | 酉
卯 | 中宫     | 戌
寅 | 丑 | 子 | 亥
```
**地支方位固定，不能改**。这是紫微的硬规则。

### 每个宫位需展示
- 宫名（如"命宫"）+ 宫位干支（如"甲午"）
- 主星（大字 + 庙旺利平陷状态小标签）
- 辅星（中字）
- 小星（小字、可弱化）
- 大限范围（如"4-13"）
- 小限 / 流年虚岁列表
- 化曜染色（禄=朱砂红、权=青蓝、科=翠绿、忌=玄黑加粗下划线）
- 自化标记（↑权 / ↓忌 等小标签）

### 特殊宫位标记
- **命宫**：双线边框 + 米黄渐变 + 右下"命"印章
- **身宫**：右上角"身"字朱砂方印
- **来因宫**：右下角"来因"小字
- **当前大限宫**：底部"大限 X-X"加粗变朱砂红

### 中宫信息
姓名 / 性别 / 阳历 / 农历 / 真太阳时 / 地经 / 阴阳 / 五行局 / 命主星 / 身主星 / 子年斗君 / 四枚干支柱印章

## 4. 模板 B：八字四柱盘 — 设计要点（待设计）

### 主区
四列（年 / 月 / 日 / 时），每列：
- 顶部：十神（如"伤官"小字）
- 上：天干（大字）
- 中：地支（中字）
- 下：藏干 + 各自十神（极小字 stack）
- 底：十二长生 / 自坐 / 纳音（一行三列）

### 大运带
横向时间轴 10 列，每列：干支 + 起运年 + 起运岁 + 十神
**当前大运高亮**（朱砂底色或左侧朱砂条）

### 流年带
当前大运下方展开 10 个流年，每个：年份 / 虚岁 / 干支 / 十神

### 算法层补字段区块
和四柱并列或下方，分卡片：
- 格局（含 confidence、notes）
- 旺衰（含分数、四维细节）
- 月令五行旺相休囚死
- 五行统计（surface + 含藏干 双柱状图）
- 调候用神
- 天干关系（合 / 克）
- 地支关系（刑冲合害 / 拱合拱会 / 暗合）
- 整柱判定（盖头 / 截脚 / 天地同气）

### 中宫风格的命主信息块
和紫微模板的中宫同款（保持气质一致）

## 5. 技术约束

- **单文件 HTML**：CSS inline；JS 可选但尽量无；无构建步骤
- **字体**：可走 Google Fonts CDN（Noto Serif SC、JetBrains Mono）+ 系统 fallback
- **打印友好**：A4 横版可以一页装下，无截断
- **桌面优先**：1080px 宽容器；手机端能看但非主战场
- **截图友好**：整版可视区域 ≤ 1920×1200，方便一次截图
- **零外部资源依赖**：除字体 CDN 外不依赖任何外链（图片用 SVG inline / data URI）

## 8. 开放讨论问题

下面这些是设计决策点，需要在迭代中拍板：

1. **字体来源**：Google Fonts CDN（Noto Serif SC + JetBrains Mono）保证渲染一致？还是允许 fallback 到系统字体接受差异？
2. **化曜的视觉权重**：现在化曜只染色，要不要再加底色块 / 角标 / 印章？强化"四化"的视觉地位
3. **煞星的可视化**：羊陀火铃地空地劫这些煞星，用红色警示还是和辅星同等处理？
4. **装饰元素的克制边界**：印章保留几枚？装裱双线还是单线？要不要加水墨晕染？
5. **空宫的处理**：主星空的宫位（如本例的财帛宫、交友宫）现在写"—"，要不要换成更有意境的标记（如"虚"印）？
6. **当前大限的视觉强调**：现在用朱砂红，要不要加底色块 + 大限编号印章？让"我现在走在哪个大限"一眼看到
7. **限流叠宫的标记**：如果出现"限流叠宫：紫破 / 武贪"，要怎么标？现在的模板没考虑这种叠加
8. **流年密度**：每个宫位列了 5 个流年虚岁，是不是太多？要不要只列 1-2 个最近的？
9. **印刷可读性**：现在的色对比度（朱砂 vs 米底）打印出来够清晰吗？需不需要"打印模式"
10. **八字模板与紫微模板的"同款气质"如何具体定义**：共用 token + 共用装饰元素 + 共用栅格逻辑？
